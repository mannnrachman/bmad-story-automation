# Contributing to BMAD Story Automation

Thank you for your interest in contributing to BMAD Story Automation! We greatly appreciate contributions from the community.

## How to Contribute

### Reporting Bugs

If you find a bug, please create a new issue with the following information:

- **Clear description** of the bug found
- **Steps to reproduce** the bug
- **Expected behavior** vs **actual behavior**
- **Environment** (OS, Python version, etc.)
- **Screenshots or logs** if relevant

### Proposing New Features

To propose a new feature:

1. Create a new issue with the `enhancement` label
2. Describe the proposed feature in detail
3. Explain why this feature is useful
4. Provide usage examples if possible

### Pull Request Process

1. **Fork** this repository
2. **Create a branch** for your feature or fix:
   ```bash
   git checkout -b feature/feature-name
   ```
3. **Commit** your changes with a clear commit message:
   ```bash
   git commit -m "Add: brief description of changes"
   ```
4. **Push** to your branch:
   ```bash
   git push origin feature/feature-name
   ```
5. **Create a Pull Request** to the `main` branch of the main repository

### Coding Guidelines

- Use **Python 3.8+** syntax
- Follow **PEP 8** style guide
- Add **docstrings** for new functions and classes
- Ensure your code **does not generate errors**
- Test your changes before submitting PR

### Commit Message Convention

Use the following format for commit messages:

- `Add:` for adding new features
- `Fix:` for fixing bugs
- `Update:` for updating existing features
- `Refactor:` for code refactoring
- `Docs:` for documentation changes

Example:

```
Add: support for custom iteration timeout
Fix: progress bar not updating on Windows
Update: improve error handling
```

### Testing

Before submitting PR, ensure:

- Script runs properly in **demo mode**
- No **errors** or **warnings**
- New features work as expected

### Code Review

All Pull Requests will be reviewed by maintainers. We may:

- Request changes or improvements
- Provide suggestions for improvement
- Merge PR if it meets standards

## Questions?

If you have questions, please:

- Create a new issue with the `question` label
- Contact maintainers through GitHub
